Notice!
As it is clearly illustrated with the green bug icons and the BETA word, this is an;
   - unstable,
   - bug-ridden,
   - not finished
version of QuArK 6, you are able to download here.

Get Python First
Before installing QuArK 5 or 6, you must have installed the Python-language if you haven't got it already. You can either download the QuArK specific mini-python package, or get the full package from the official Python site:

  minipy15b.exe
or
  www.python.org

How to Install
It is recommended that you install (unzip) this QuArK 6 Beta-1 in a different folder from any existing QuArK installation you may have!

Asking for Help
If you have any questions/problems regarding this QuArK 6 Beta-1, please send or mail them to the QuArK forum at www.eGroups.com. You can also subscribe to the QuArK forum mailing list which makes it easier to keep in touch, and to follow replies.

BETA means:   "Under Construction" 
New things have been introduced to QuArK 6 Beta-1, and some of them don't work as of yet:
  • Bezier editing is very limited; cannot resize the "quilts" (eq. add/remove handles), nor position the texture. (To apply a texture, open the texture browser from the "Toolboxes" menu.)
  • Reading Q3A shader definitions from .SHADER files is implemented, but it is not possible to view/edit a shaders definition.
These things should work:
  • Bezier editing, though limited to only the "quilt"-handles (3x3) that the bezier-object from the UserData panel will create.
  • The OpenGL 3D-window should now correctly stay "On Top" of other windows.
  • Automatic extraction of texture-files and generation of a shader-file for Q3MAP.EXE to .\tmpQuArK in the "Quake III Arena"-folder.
  • In the texture-browser, viewing the first image a shader definition referes to.
Getting QuArK 6 Beta-1
There are different compilations of QuArK 6 Beta-1, but you only have to download one of them.

"With debug" will catch more bugs, but runs a little bit slower than "Without debug". It's better to help us fix bugs, and safer for you (as it will rather catch bugs when they occur, instead of letting the bug corrupt your map).

When you discover a bug, that has not already been mentioned in the QuArK-Python forum, please write a message to the forum, with a clear and understandable description of the problem. You are welcome to upload any pictures and other documentation, regarding the found bug, to the QuArK-Python forum's Files-vault.

Att. other webmasters: Please link to the Official QuArK homepage and not the files directly! That will ensure that people always know where to get the newest and updated files for QuArK, and not some 6+ months old obsolete copy.
